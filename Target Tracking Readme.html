Target Tracking Example

This example shows how to track a target in an image based on a template. You can select from three target templates to track. You also can modify the example to run with a live camera, track other targets, or track more targets.

Note  You can modify this example to work with different hardware targets. The loop in this example has adjustable delays. However, keep in mind that various factors control the the actual loop period.

Example Requirements:

  • Vision Acquisition Software
  • NI-IMAQ device drivers

Changing the Target to Track

Complete the following steps to configure the target the program tracks.

  1. From the block diagram of Target Tracking.vi, double-click the Vision Assistant VI.
  2. Select File»Open Image and browse to the test image you want to load.
  3. Double-click the Pattern Matching step in the script.
  4. Click the Template tab.
  5. Click the New Template button.
  6. In the Select Template Region screen of the NI Vision Template Editor, click the image at the corner of the target object and drag the cursor to surround the target.
  7. Click the Next button.
  8. In the Define Pattern Matching Mask screen, use the tools in the Draw Template Regions to Ignore to select any regions of the target image to ignore.
  9. Click the Finish button to save the template and exit the NI Vision Template Editor.
  10. Click the OK button on the Template tab of the NI Vision Assistant.
  11. Click the Finish to return to LabVIEW.
  12. Update the Template File Path input of the Vision Assistant Express VI with the path to the new template file.

Timing Considerations for Vision Examples

Although you can adjust the amount of delay of the While Loop in the Robotics vision examples, the following factors control loop periods of vision applications:

  • Time required to acquire an image—The time required to get an image from the acquistion depends on the frame rate of the camera, time required to transfer the image, and time required to decompress the image, if necessary.
  • Time required to process the image—The time required to process the image depends on the algorithms you use. Many vision algorithms are not deterministic because the execution time is data dependent. You should test with a variety of scenes to accurately measure the jitter if processing time is important for your application. Select Tools»Performance Meter in the NI Vision Assistant to benchmark each step of the image processing.
  • Any specific timing constraints—Implement the timing behavior you want using the timing constraint options in LabVIEW, such as the Wait (ms) and Wait Until Next ms Multiple functions and Timed Loops.

Using Hardware Targets with Vision Examples

This example runs on desktop computers. You can modify this example to work with different hardware targets, such as CompactRIO devices and NI Smart Cameras.

cRIO Real-Time Controller

Complete the following steps to configure this VI to run on a cRIO real-time controller.

  1. Open the .lvproj file.
  2. Add the RT target you want to use in the Project Explorer window.
  3. Drag the .vi file from My Computer to the RT target.
  4. (Optional) If you want to run a simulated image acquistion you must transfer the image files to the RT target. You can use an FTP utility such as the utility in the National Instruments Measurement & Automation Explorer (MAX) to transfer files. Complete the following steps to transfer image files to the RT target in MAX.
    1. In MAX, right-click the target in the configuration tree and select File Transfer from the shortcut menu to launch the File Transfer Utility.
    2. Create a folder labeled images in the /ni-rt/startup/ directory.
    3. Copy the .png files into this images folder. You can find the location of the files on the desktop computer by opening the Files tab of the Project Explorer window.
    4. Exit the File Tranfer Utility.
  5. Open the .vi file under the RT target and click the Run button to deploy and run the VI on the RT target.

NI Smart Camera

Complete the following steps to configure this VI to acquire live images from an NI Smart Camera.

  1. From the block diagram of the .vi file, double-click the Vision Acquistion Express VI to launch the NI Vision Acquisition Express dialog box.
  2. Use the Back and Next buttons to navigate to the Select Acquisition Source page.
  3. Select the camera from the Acquisition Sources for Localhost list.
  4. Click the Finish button to exit the NI Vision Acquisition Express dialog box.
  5. Delete the broken wire that passes a file path to the Image Path input of the Vision Acquisition Express VI.
